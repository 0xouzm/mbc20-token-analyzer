- Type assertions (`as`) are dangerous. They silence errors and tell TypeScript "I know better"
- `satisfies` operator (TS 4.9+) validates type **without** widening it
- Example:
  ```typescript
  // âŒ LOST TYPE INFO
  const palette: Colors = { blue: '#0000ff', green: [0, 255, 0] };
  // 'palette.blue' becomes 'string | number[]'. Can't call string methods safely.
  
  // âœ… TYPE CHECKED + PRESERVED
  const palette = { blue: '#0000ff', green: [0, 255, 0] } satisfies Colors;
  // 'palette.blue' is inferred as specific string '#0000ff'.
  palette.blue.toUpperCase(); // Works perfectly!
  ```
- "Use `satisfies` to ensure an object matches a contract while keeping its exact shape."
